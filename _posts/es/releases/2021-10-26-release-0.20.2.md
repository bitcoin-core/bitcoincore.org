0.20.2 Notas del lanzamiento
============================

Bitcoin Core versión 0.20.2 ya está disponible en:

  <https://bitcoincore.org/bin/bitcoin-core-0.20.2/>

Esta actualización menor incluye varios arreglos de errores y mejoras
en el rendimiento, así como traducciones actualizadas.

Por favor informe de errores usando la aplicación de seguimiento de
cuestiones en GitHub:

  <https://github.com/bitcoin/bitcoin/issues>

Para recibir notificaciones sobre actualizaciones y sobre seguridad (en
inglés), por favor suscríbase a:

  <https://bitcoincore.org/en/list/announcements/join/>

Cómo actualizar {#how-to-upgrade}
=================================

Si está vd. utilizando una versión antigua, apáguela. Espere a que se
haya detenido por completo (lo cual podría tardar varios minutos en
algunos casos), entonces ejecute el instalador (en Windows) o sin más
copie o arrastre `/Applications/Bitcoin-Qt` (en macOS) o
`bitcoind`/`bitcoin-qt` (en GNU/Linux).

Actualizar directamente desde una versión de Bitcoin Core que haya
alcanzado su fin de vida útil (EOL) es posible, pero podría tardar
algo de tiempo si el directorio de datos requiere ser migrado. Las
versiones antiguas de la cartera de Bitcoin Core están por lo general
soportadas.

Compatibilidad {#compatibility}
===============================

Bitcoin Core tiene soporte para y es exhaustivamente probado en los
sistemas operativos que utilizan el kernel Linux, macOS 10.12+, y
Windows 7 y posteriores.  Bitcoin Core debería además funcionar en la
mayoría de los otros sistemas UNIX-like pero no está tan
frecuentamente comprobado en ellos.  No es recomendable usar Bitcoin
Core en sistemas no soportados.

Desde Bitcoin Core 0.20.0 en adelante, las versiones de macOS anteriores
a la 10.12 ya no están soportadas. Adicionalmente, Bitcoin Core
todavía no cambia de apariencia cuando el "modo oscuro" ("dark mode")
está activo.

Errores conocidos
=================

El proceso para generar la edición del código fuente ("tarball") ha
cambiado, en un esfuerzo por hacerlo más completo, sin embargo, hay
varios fallos de regresión en esta edición:

- El script `configure` generado está desaparecido, y necesitarás
  instalar Autotools y ejecutar `./autogen.sh` antes de poder ejecutar
  `./configure`. Esto ocurre igual cuando el código es obtenido usando
  Git.

- En lugar de ejecutar simplemente `make`, deberás ejecutar en su lugar
  `BITCOIN_GENBUILD_NO_GIT=1 make`.

Cambios notables
================

Cambios concernientes a iguales malvados (misbehaving peers)
------------------------------------------------------------

Iguales que se comportan mal (p.ej. nos envían bloques inválidos)
ahora están reseñados como nodos desaprobados en la salida de log, ya
que no son (ni eran) estrictamente prohibidos (banned): se siguen
aceptando sus conexiones entrantes, pero son preferidos para desalojo.

Además, varios cambios adicionales se han introducido al cómo las
direcciones desaprobadas son tratadas:

- Desaprobar una dirección no caduca automáticamente a las 24 horas (o
  tu configuración de `-bantime` correspondiente). Dependiendo del
  tráfico proveniente de otros iguales, la desaprobación podrá
  caducar en un momento indeterminado.

- La desaprobación no tiene persistencia entre reinicios.

- No hay un método para listar las direcciones desaprobadas. No las
  retorna la llamada a procedimiento remoto (RPC) `listbanned`. Además
  esa llamada a procedimiento remoto ya no informa el campo
  `ban_reason`, ya que `"manually added"` es la única opción que
  queda.

- La desaprobación no se puede quitar con el comando RPC
  `setban remove`. Si necesitas quitar una desaprobación, puedes quitar
  todas las desaprobaciones deteniendo y encendiendo tu nodo.

Cambios en notificaciones
-------------------------

Ahora se envían notificaciones `-walletnotify` para transacciones de la
cartera que se quiten de la mempool porque hagan conflicto con un nuevo
bloque. Estas notificaciones se enviaban previamente antes de la
edición v0.19, pero han estado rotas desde esa edición (error #18325).

Cambios en las transacciones parcialmente firmadas (PSBT)
---------------------------------------------------------

Las transacciones parcialmente firmadas (Partially Signed Bitcoin
Transaction, PSBT) contendrán tanto la UTXO no witness como la UTXO
witness para entradas segwit, para restaurar la compatibilidad con el
software de cartera que ahora necesita la transacción previa entera
para entradas segwit. La UTXO witness se sigue proveyendo para mantener
la compatibilidad con software que se apoyaba en su existencia para
determinar si una entrada era segwit.
